# Conrad Seoul 상품권 & 증서 관리 대시보드(Certificate Ledger) 사용자 매뉴얼

본 매뉴얼은 Conrad Seoul의 **상품권 및 서비스 증서(Gift & Service Certificate)**를 효율적이고 투명하게 관리하기 위해 구축된 대시보드의 기능별 사용 방법임.

---

## 1. 상품권 판매 및 신규 등록 (New Issue)
상품권이나 서비스 증서를 판매했을 때 시스템에 등록하는 프로세스임.

* **메뉴 진입**: 좌측 메뉴 `증서 목록` 선택 → 우측 상단 `+ 새 증서 발행` 버튼 클릭
* **발행 모드 선택**:
  * **연번 여러 장 (Bulk)** `[권장]`: 동일 종류의 연속 번호 대량 등록 시 사용함.
    * 종류, 서비스 포함내역 선택 후 **시작 증서번호** 및 **수량** 입력 (우측에 예상 번호 대역 실시간 표시됨).
    * 금액, 발행일, 결제수단, 비고 입력 후 `생성` 클릭 → 하단 목록에 자동 생성됨.
  * **1장씩 발행 (Single)**: 수동으로 개별 입력하여 등록함.
* **최종 등록**: 목록 내용 검토 후 하단 `전체 등록` 버튼을 눌러 저장함.

> [!NOTE]
> * **유효기간(만료일) 자동 계산**: 발행일 선택 시 종류별 유효기간 자동 지정됨 (서비스 1년, 상품권 5년).
> * **증서번호 자동 제안**: 번호 입력 칸에서 `Tab` 키를 누르면 시스템에 등록된 마지막 번호 다음 번호가 자동으로 제안 입력됨.

---

## 2. 상품권 사용 처리 (Redemption)
고객이 업장에 방문하여 증서를 사용했을 때 매출로 인식하는 과정임.

* **메뉴 진입**: `증서 목록`에서 대상 증서 검색 후 체크박스 선택 → 상단 툴바 `사용 처리` 클릭
* **정산 유형별 입력**:
  * **정상 사용**: 사용일 입력 및 면가(A)와 동일하게 **매출금액(B)** 적용 (잡이익 C는 0원).
  * **만료 후 사용 (Late Use)**: 유효기간 경과한 서비스 증서 사용 시 **매출(B) 90% / 잡이익(C) 10%** 자동 분할됨. **매출등록일** 입력 필수임.
  * **잔액 환급 (금액권)**: 일부 사용 후 잔액 현금 반환 시, 사용액을 **매출금액(B)**에, 잔액을 **환불액(D)**에 입력함.
    * 60% 이상 사용 시 잔액 환급 대상이 됨. (공식: 면가 A = 매출 B + 잡이익 C + 환불액 D 이 되어 차액이 0원이어야 함)
* **비고 입력**: 영수증 번호 또는 객실 번호 등 식별 값을 **비고(Bill No.)**에 기입함.
* **최종 완료**: `일괄 처리 / 사용 처리` 버튼 클릭하여 저장함.

---

## 3. 취소 및 환불 처리 (Void & Refund)
발행 오류로 증서를 파기하거나 판매된 증서를 반품/환불할 때 사용함.

* **메뉴 진입**: `증서 목록`에서 ACTIVE(유효) 증서 선택 → 상단 툴바 `취소/환불` 클릭
* **사유 선택**:
  * **미판매 취소 (Misprint)**: 오발행 파기용. 금액 이동 없으므로 환불액 0원 처리함.
  * **판매 후 환불 (Refund)**: 실판매 건 환불 처리함.
    * **전액 환불**: 면가 전체를 **환불액(D)**으로 처리함.
    * **위약금 10% 차감 환불**: 면가의 10%는 **잡이익(C)**으로 계상하고, 90%만 **환불액(D)**으로 지급 정산함 (미리보기로 확인 가능).
* **최종 완료**: 환불일 선택 후 `일괄 처리` 클릭하여 완료함.

---

## 4. 유효기간 만료 및 소멸 인식 (Expiry & Write-Off)
유효기간이 경과한 증서들을 잡이익 계정으로 일괄 마감(Write-off) 처리하는 작업임 (주로 분기/연말 회계 마감 시 수행).

* **메뉴 진입**: 좌측 메뉴 `만료 대기열` 선택
* **현황 확인**: 상단 FORECAST 테이블에서 올해 누적 전환, 대기 중, 향후 만료 예정 현황 확인 가능함.
* **소멸 마감 처리**:
  * **선택 처리**: 특정 건 선택 후 `선택 항목 인식 처리` 클릭.
  * **일괄 처리**: `전체 연말 일괄 인식 처리 (Year-End)` 클릭 → 대기 중인 모든 만료 건 100% **잡이익(C)** 계정으로 일괄 마감됨 (상태는 '잡이익 전환'으로 변경됨).

---

## 5. 잡이익 환원 처리 (Grace Use)
이미 유효기간이 지나 연말 잡이익으로 100% 마감 처리된 **서비스 증서(Service Certificate)**에 대해, 고객이 소멸시효(발행일로부터 5년) 이내에 찾아와 예외적으로 사용을 요청하는 경우 적용하는 소급 처리임.

* **처리 방법**: `증서 목록`에서 대상 건 선택 (상태가 '잡이익 전환'이어야 함) → 상단 툴바 `잡이익 환원` 클릭 → 고객 사용일 입력 후 완료함.
* **회계 처리 결과**:
  * 기존 100% 잡이익(C) 중 **90%는 매출(B)로 반제 전환**, **10%만 잡이익(C)으로 최종 확정**됨.
  * 원장(Misc Revenue)에 환원 지급 및 매출 반제 내역 자동 기록되며, 상태는 '잡이익 환원'으로 변경됨.

> [!WARNING]
> * **Gift 상품권**: Gift 상품권은 예외 규정이 없으므로 만료 전환 완료 후 환원이 불가능함.
> * **5년 만료 경과**: 발행일로부터 5년이 완전히 지난 서비스 증서는 법적 소멸시효 완성으로 환원이 불가능함.

---

## 6. 유효기간 연장 (Extend Expiry)
부득이한 사유로 만료 전에 유효기간을 연장해주어야 할 때 사용함.

* **처리 방법**: `만료 대기열` 또는 `증서 목록`에서 대상 건 우측의 `유효기간 연장` 클릭.
* **필수 입력**: **새 만료일** (미래 날짜) 및 **Mate Approval #** (관리자 승인번호) 입력 후 `연장` 클릭함.
* **결과**: 만료 대기열에서 제외되며 다시 ACTIVE(유효) 상태로 복원됨.

---

## 7. 정합성 점검 및 부가 기능
시스템 안정성과 데이터 오류 방지를 위한 핵심 기능임.

* **정합성 점검 (Integrity Check)**: 
  * 회계 오류(차액 불일치, 날짜 누락 등) 실시간 감지함.
  * 오류 항목 우측의 `수정 (Fix)` 클릭 시 해당 증서 잠금 해제되어 즉시 수정 가능함.
* **작업 되돌리기 (Undo)**:
  * 대시보드 상단 정보바의 `되돌리기` 버튼으로 직전 작업(발행, 사용, 취소 등) 1단계 즉시 복구함.
* **이력 관리 및 보안**:
  * 모든 데이터 변경은 전후 스냅샷과 함께 `감사 로그 (Audit Log)`에 영구 기록됨.
  * 수동 수정 시 `변경 사유` 기입 필수이며, 부서 공용 로그인으로 보안 유지함.
* **가져오기/내보내기**:
  * 엑셀 원장 업로드 및 내보내기 지원함. 업로드 오류 건은 '검토 필요'로 자동 분류됨.
